# OTA GUI 验收标准 Checklist

## 基础功能 (Basic Functions)

- [ ] **BF-01** 程序启动正常，窗口标题显示 "Car Panel OTA Tool v1.0"
- [ ] **BF-02** 窗口尺寸 720x680，不可调整大小
- [ ] **BF-03** 点击"刷新"按钮可枚举系统 COM 端口
- [ ] **BF-04** 波特率下拉框包含预设值，默认 115200
- [ ] **BF-05** 点击"浏览..."可打开文件选择对话框，筛选 .bin 文件
- [ ] **BF-06** 选择 .bin 文件后显示文件名、大小、修改时间
- [ ] **BF-07** 选择 5 分钟前生成的文件时，显示旧文件警告

## 拖拽功能 (Drag & Drop) — 依赖 tkinterdnd2

- [ ] **DD-01** 从文件管理器拖拽 .bin 文件到文件选择区域，路径自动填充
- [ ] **DD-02** 拖拽非 .bin 文件时弹出"仅支持 .bin 固件文件"警告
- [ ] **DD-03** 拖拽的 .bin 文件信息正确显示（大小、时间）

## OTA 升级流程 (OTA Flow)

- [ ] **OTA-01** 未选择 COM 端口点"开始升级"时弹出错误提示
- [ ] **OTA-02** 未选择文件点"开始升级"时弹出错误提示
- [ ] **OTA-03** 点击"开始升级"后弹出确认对话框，显示文件信息
- [ ] **OTA-04** 确认后 UI 组件禁用（端口/文件/开始按钮不可用）
- [ ] **OTA-05** "取消升级"按钮变为可用
- [ ] **OTA-06** 进度条从 0% 开始更新，显示百分比和 KB 进度
- [ ] **OTA-07** 日志区域显示实时日志（带时间戳）
- [ ] **OTA-08** 窗口不卡顿（后台线程运行 YMODEM 协议）

## 端口占用检测 (Port Conflict)

- [ ] **PC-01** 串口被其他程序占用时，弹窗提示具体错误信息
- [ ] **PC-02** 串口打开失败后，UI 恢复可用状态

## 取消功能 (Cancel)

- [ ] **CA-01** 点击"取消升级"后，传输被中断
- [ ] **CA-02** 取消后 UI 恢复可用状态
- [ ] **CA-03** 取消后串口正确关闭

## 芯片信息 (Chip Info) — Phase 2

- [ ] **CI-01** "获取芯片信息"按钮可见可用
- [ ] **CI-02** 点击后发送查询指令，等待 MCU 响应
- [ ] **CI-03** 收到响应后芯片信息面板更新（MCU 型号、分区、版本等）
- [ ] **CI-04** 超时或失败时有合理提示

## 日志管理 (Log Management)

- [ ] **LM-01** 日志最多显示 5000 行，超出自动裁剪
- [ ] **LM-02** "清空日志"按钮清空日志文本框
- [ ] **LM-03** "保存日志"按钮将日志保存为 .txt 文件
- [ ] **LM-04** 保存日志失败时弹窗提示

## 配置检查 (Config Check) — 若文件不存在则使用默认值

- [ ] **CC-01** ui_config.json 缺失时程序正常启动，使用默认参数
- [ ] **CC-02** 修改 ui_config.json 中窗口标题/尺寸/颜色后，重启生效

## UI 自定义验证 (UI Customization)

- [ ] **UC-01** 修改 chip_panel.show 为 false 可隐藏芯片信息面板
- [ ] **UC-02** 修改 log_panel.height 可调整日志行数
- [ ] **UC-03** 修改 colors 配色方案可改变 UI 外观
- [ ] **UC-04** 修改 window.title 可改变窗口标题

## 打包 (Packaging)

- [ ] **PK-01** 双击 preview.bat 可在无打包情况下预览 GUI
- [ ] **PK-02** 双击 build.bat 可一键打包出 CarPanel_OTA_Tool.exe
- [ ] **PK-03** 打包后的 exe 可脱离 Python 环境独立运行
